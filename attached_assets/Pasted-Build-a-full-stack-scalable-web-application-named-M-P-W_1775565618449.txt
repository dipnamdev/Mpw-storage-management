Build a full-stack scalable web application named:

"M.P. Warehousing & Logistics Corporation"

Tech Stack:

* Frontend: React (JavaScript) + Vite + TailwindCSS
* Backend: FastAPI (Python)
* Database: PostgreSQL
* ORM: SQLAlchemy (async preferred)
* Auth: JWT-based authentication
* Email: Use SMTP (mock allowed but structure should support real email sending)
* File Upload: Support image upload (for remark screenshot)

---

CORE REQUIREMENTS:

There are 2 roles:

1. Admin
2. Branch Operator

---

DATABASE DESIGN (IMPORTANT: FOLLOW STRICTLY)

User:

* id (PK)
* name
* email (unique)
* password_hash
* role (admin/operator)
* branch_name
* district_name
* mobile_number
* created_at

Commodity:

* id (PK)
* crop_name
* crop_year
* per_bag_per_month
* UNIQUE(crop_name, crop_year)

Depositor:

* id (PK)
* name
* gst_no
* total_gst

Bill (MAIN TABLE):

* id (PK)

* serial_no (AUTO-INCREMENT, GLOBAL, UNIQUE)

* created_by (FK → User)

* district

* branch_name

* godown_name

* bill_no

* commodity_id (FK)

* crop_year

* financial_year

* month_year

* rate_per_bag

* opening_balance

* closing_balance

* received_bags

* issue_bags

* reserve_bags

* chargeable_bags

* total_charge (formula: received_bags * commodity.per_bag_per_month / 2)

* status (pending/approved/rejected)

* is_locked (true after approval)

* created_at

BillVersion (IMPORTANT FOR EDIT FLOW):

* id (PK)
* bill_id (FK)
* data_json (store edited fields snapshot)
* status (pending/approved/rejected)
* created_by
* created_at

BillApproval:

* id (PK)
* bill_id (FK)
* depositor_id (FK)
* pass_amount
* payment_method
* neft_no
* remark
* remark_image_url
* approved_by
* approved_at

Notification:

* id (PK)
* user_id (FK)
* title
* message
* type (email/web)
* is_read
* created_at

BUSINESS RULES:

* Once bill is approved → is_locked = true → cannot be edited
* Serial number must be global and consistent
* Commodity dropdown comes from admin-created commodities
* Depositor only linked during approval
* Edit requests DO NOT overwrite original bill until approved
* Maintain audit-safe structure

* Bill Created → notify Admin

* Edit Approved/Rejected → notify Operator
